QAid Recorder — Privacy Policy

Last updated: February 13, 2026

Summary: QAid Recorder captures browser interactions solely to generate automated tests. All recorded data is sent exclusively to your self-hosted QAid server. We do not collect, store, or share your data on any third-party or external servers.

1. What This Extension Does

QAid Recorder is a browser extension that records your interactions with web pages (clicks, form fills, scrolls, navigation) and sends them to your QAid server instance to generate automated end-to-end Playwright tests.

2. Data We Collect

The extension collects data only while you are actively recording (recording must be explicitly started by the user). The following data is captured during a recording session:

User Activity

  • Clicks, double-clicks, and hovers on page elements
  • Text typed into form fields (input values)
  • Dropdown selections and checkbox/radio toggles
  • Scroll positions and directions
  • File upload actions

Website Content

  • DOM element metadata: tag names, text content, ARIA labels, placeholder values, CSS selectors, and HTML attributes of interacted elements
  • This metadata is used to generate stable test selectors — full page content is not captured

Web History

  • URLs of pages visited during the recording session
  • Navigation events (page loads, redirects) that occur while recording

Authentication Information

  • API key: A server-generated key you configure to authenticate the extension with your QAid server
  • Session cookies: When recording on login-protected sites, session cookies may be obtained from your QAid server and set on your browser to enable authenticated recording

3. Data We Do NOT Collect

  • Personally identifiable information (name, email, address)
  • Browsing activity outside of active recording sessions
  • Passwords or login credentials (session cookies are handled; raw passwords are not)
  • Financial or payment information
  • Location or geolocation data
  • Health information

4. Where Data Is Stored

  • Locally: Recording state and extension settings are stored in chrome.storage.local on your device. This data persists between browser sessions and is cleared when the extension is uninstalled.
  • Your QAid server: When you save a recording, the captured actions are sent to your self-hosted QAid server instance. The server URL is configured by you.
No external servers: Recorded data is never sent to QAid's servers, third-party analytics services, or any destination other than the QAid server URL you configure.

5. How Data Is Used

All collected data is used exclusively to:

  • Generate automated Playwright test scripts from your recorded interactions
  • Resolve element selectors for reliable test targeting
  • Maintain recording state across browser sessions

6. Data Sharing

We do not sell, transfer, or share user data with third parties. Recorded data is transmitted only to the QAid server URL that you configure.

7. Data Retention

  • Local storage: Recording data is cleared when you finish and save a recording, start a new recording, or uninstall the extension.
  • Server storage: Data retention on your QAid server is governed by your own server configuration and policies.

8. Your Controls

  • Recording only starts when you explicitly click the record button
  • You can delete individual recorded actions before saving
  • You can stop recording at any time
  • Uninstalling the extension removes all locally stored data

9. Permissions

The extension requests browser permissions necessary for its recording functionality. Each permission and its justification are listed on the Chrome Web Store listing. No permissions are used for purposes beyond test recording.

10. Changes to This Policy

We may update this privacy policy to reflect changes in the extension's functionality. The "Last updated" date at the top will be revised accordingly. Continued use of the extension after changes constitutes acceptance of the updated policy.
